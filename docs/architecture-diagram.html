AgenticWeb System Architecture

Multi-channel autonomous web-agent platform using React, FastAPI, LangGraph, MCP-style tools, Playwright, provider fallback, and SQLite memory.
Clients Frontend Gateway Agent Runtime Tools Persistence

Primary Runtime Flow

1. GoalUser submits a goal from web UI, extension, Telegram, or REST/SSE.
2. SessionGateway or agent API assigns session id, provider, active task, and cancellation path.
3. Memoryrun_agent creates SQLite session and recalls relevant session-scoped memory.
4. ACTLangGraph prompts the selected provider with tools and compact context.
5. OBSERVEToolNode executes search, scrape, browser, or extraction calls.
6. StreamStatus, step logs, and browser screenshots stream back as JSON events.
7. FinalSUMMARISE writes a grounded answer, stores summary, and emits done.

Two Backend Paths

  • Web UI and Telegram enter through gateway/main.py.
  • Chrome extension calls agent/server.py directly on :8765.
  • Both paths use the same run_agent() runtime.

State Boundaries

  • Gateway sessions are process-local.
  • LangGraph state is per run.
  • SQLite survives restarts; active tasks do not.

Streaming Events

  • status: progress message.
  • step: tool result snippet.
  • canvas: base64 browser screenshot.
  • done/error/cancelled: terminal states.
